# PatchLens Roadmap

## v0.1.0 — Prototype Baseline

Status: complete

- Mobile-first project dashboard
- Versioned test sessions
- Release-readiness checklist
- Evidence-backed defect capture
- Regression status ledger
- Markdown and Codex prompt output
- JSON portability and local persistence
- PWA app-shell caching

## v0.1.1 — Hardening

- Add explicit session selection outside the report screen
- Add delete/archive controls with confirmation
- Add storage-usage indicator and screenshot compression
- Add keyboard and screen-reader audit
- Add broader browser automation for import/export and completed sessions

## v0.2.0 — Test Templates

- Custom checklist templates by project
- Reusable device/browser profiles
- Project-specific guardrails
- Session duplication for regression retests
- Defect linkage across versions

## v0.3.0 — Evidence Tools

- Screenshot markup and crop tools
- Side-by-side expected/actual evidence
- Attach multiple screenshots
- Export a compact evidence bundle

## v0.4.0 — GitHub Bridge

- Explicit repository authorization
- Preview GitHub issue payload before submission
- Create issues only after user confirmation
- Preserve PatchLens session and defect references
- Never push code or modify repositories automatically
